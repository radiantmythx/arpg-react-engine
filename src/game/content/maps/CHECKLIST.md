# Map Definition Checklist

- Use a unique `id` and human-readable `name`.
- Confirm `tier` and `unlockReq` fit the campaign sequence.
- Point to valid layout, encounter, and reward profile ids.
- Keep the description short and biome-focused.
- Preserve runtime parity by changing only authoring files until the map is ready to migrate.
- Validate encounter pressure and rewards before registry integration.
