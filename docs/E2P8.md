# E2 Phase 8 — Ring-by-Ring Theming & Content Pass

This document tracks every sub-phase of E2P8: one phase per ring (or cluster).
Each phase is tested in-game before moving to the next.

---

## Design Rules (apply to every phase)

1. **Minor nodes** do not need unique names — generic numbered labels are fine.
2. **Minor nodes** follow **one thematic format** per branch (e.g. all fire damage, all health regen — not a mix).
3. **Branches must end with a Notable or Keystone** — no dead-end minor as the final node.
4. **Highway rings (r3, r7, r10, r15) contain NO notables or keystones** — travel only.
5. **Notables / Keystones only at the END of a branch** — never in the middle.
6. **Perfect tri-radial symmetry** — every slot pattern in one class section must mirror the other two at +120° / +240° offsets with comparable stat budgets.

### Ring type reference

| Ring | Type | Arc behaviour |
|------|------|---------------|
| r0 | Hub (8 slots) | Isolated starting cluster |
| r1 | Inner (16 slots) | Connects hub to class gates |
| r2 | Class Gates (36 slots) | Start nodes for each class |
| r3 | **Highway** | Fully arc-connected, travel only |
| r4 | Broken | 6 dead-end branches (6 nodes each) |
| r5 | Broken | " |
| r6 | Broken | " |
| r7 | **Highway** | Fully arc-connected, travel only |
| r8 | Broken | " |
| r9 | Broken | " |
| r10 | **Highway** | Fully arc-connected, travel only |
| r11 | Broken | " |
| r12 | Broken | " |
| r13 | Broken | " |
| r14 | Broken | " |
| r15 | **Terminal Highway** | Fully arc-connected, travel only |

### Branch anatomy (broken rings, 6 nodes per branch)

```
s00 (spoke, entry)
├─ arc → s01 (minor)
├─ arc → s02 (minor)
├─ arc → s03 (minor)
├─ arc → s04 (minor)
└─ arc → s05 ← MUST be Notable or Keystone (dead-end cap)
```

Same pattern repeats for branches starting at s06, s12, s18, s24, s30.

### [x] E2P8.7.5 — r4a / r4b Spur Branches + Ring Spacing Expansion

**Scope:** Two new pseudo-rings (`r4a`, `r4b`) — each holding 6 short defense-themed branch stubs
sprouting off the 6 connector (entry-spoke) nodes in r4. Plus a global renderer change to double
inter-ring spacing so the tree breathes more.

---

#### Architecture Overview

```
     r3s00  (highway)
    │ spoke
    |               r4a  ←── new inner branch (4 nodes + notable)
                  │ 
   r4s00  ────────●────────  r4s01 … r4s05 (4 nodes + notable dead end)
  (connector)     │ 
                   r4b  ←── new outer branch (4 nodes + notable)
    │ spoke
    - r5s00  
```

- **r4a** sits radially **between r3 and r4** (1/3, same angular slot as the connector).
- **r4b** sits radially **between r4 and r5** (1/3, same angular slot as the connector).
- Each connector node (`r4s00`, `r4s06`, `r4s12`, `r4s18`, `r4s24`, `r4s30`) gains **2 new connections**: one inward to `r4a` and one outward to `r4b`.
- r4a and r4b nodes are **dead-end stubs** — no arc connections to neighbours, no onward spokes.
- A stub branch is **5 nodes long** (4 minors + 1 notable at the terminus), extending **away from the connector** along the arc direction (clockwise matching r4).

---

#### Visual / Renderer Change

**Double inter-ring spacing** in the renderer config (e.g. `RING_RADIUS_STEP` or equivalent):
- Current spacing sets rings at `ring × step` pixels from center.
- New: `ring × step × 2` — gives more room for spur sub-rings and avoids node crowding as the tree grows toward r15.
- r4a rendered at `r4_radius − third_step`; r4b at `r4_radius + third_step`.
- Spur nodes on r4a/r4b are rendered at the **same angular slot** as their parent connector.

- [x] **Renderer:** double inter-ring `RING_RADIUS_STEP`; add r4a / r4b radius positions to ring-draw loop
- [x] **passiveTree.js:** 30 new r4a nodes (6 branches × 4 nodes + 1 notable), branch connects to r4 connector
- [x] **passiveTree.js:** 30 new r4b nodes (6 branches × 4 nodes + notable), branch connects to r4 connector
- [x] **r4 connectors updated:** r4s00/06/12/18/24/30 each gain 2 new connection entries (to r4a and r4b entry nodes)
- [x] 12 Notable caps confirmed (1 per branch)
- [x] branch entry nodes rendered at correct radii (between the right ring pair)
- [x] Build passes + visual spot-check — confirm spurs visually attach to connectors
